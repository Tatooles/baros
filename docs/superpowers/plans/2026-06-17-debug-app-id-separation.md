# Debug App ID Separation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let Kevin keep the production/TestFlight Lifting Log app installed for daily use while also installing a separate Xcode Debug app for development on the same physical iPhone.

**Architecture:** Use one production app identity and one development app identity. Release/TestFlight/App Store keeps `com.kevintatooles.LiftingLog` and points at production Clerk/Convex; Debug uses `com.kevintatooles.LiftingLog.dev` and points at the existing development Clerk/Convex environments.

**Tech Stack:** XcodeGen `project.yml`, Xcode build settings, Swift unit tests, Clerk Native Applications, Apple Developer App IDs, iOS Associated Domains.

---

## Current Decision

- Production/TestFlight/App Store app:
  - Bundle ID: `com.kevintatooles.LiftingLog`
  - Display name: `Lifting Log`
  - Clerk: production
  - Convex: production
- Debug/Xcode app:
  - Bundle ID: `com.kevintatooles.LiftingLog.dev`
  - Display name: `Lifting Log Dev`
  - Clerk: existing development instance
  - Convex: existing development deployment
- No staging Clerk instance.
- No staging Convex deployment.
- No new production domain.

## File Structure

- Modify `project.yml`
  - Owns XcodeGen source of truth for build settings.
  - Add Debug bundle id and display name.
  - Keep Release bundle id and display name production-safe.
- Regenerate `LiftingLog.xcodeproj/project.pbxproj`
  - Generated by `xcodegen generate`.
  - Commit generated changes with `project.yml`.
- Modify `LiftingLogTests/ClerkConfigurationTests.swift`
  - Add Debug-hosted test coverage that proves the app under test has the dev bundle id and dev display name.
- No Convex code change
  - Debug already points at `https://glad-cow-603.convex.cloud`.
  - Release already points at `https://sensible-reindeer-16.convex.cloud`.
- No Clerk Swift code change expected
  - Clerk publishable key and associated domain already come from Info.plist build settings.
